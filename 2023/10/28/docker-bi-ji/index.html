Docker笔记

Docker为什么会出现

问题：开发–上线 ，两套环境，配置麻烦。

传统：开发jar,运维部署。

现在：开发打包部署上线，一套流程做完。

解决方案：项目带上环境安装打包。

Docker官网: https://www.docker.com/

Docker文档：https://docs.docker.com/

Docker仓库：https://hub.docker.com/

比较Docker和虚拟机技术

传统虚拟机：虚拟出一条硬件，运行一个完整的文件系统，然后在这个系统上安装和运行文件。

docker:容器内的应用直接运行在宿主机的内容，容器是没有自己的内核的，也没有虚拟我们的硬件，轻便。

每个容器间相互隔离，每个容器内都有属于自己的文件系统，互不影响。

Docker基本组成

镜像（image）

docker镜像就好比一个模板,可以通过模板创建容器服务。tomcat镜像 —>run—>tomcat容器，通过镜像可以创建多个容器。

容器（container)

Docker利用容器技术，独立运行一个或者一个组应用，通过镜像来创建的。

启动，停止，删除，基本命令！

目前可以把容器理解为一个建议的linux系统。

仓库（repository）

仓库是存放镜像的地方！

仓库分为公有仓库和私有仓库！

Docker在linux安装

#1.卸载旧版本
sudo yum remove docker \
                  docker-client \
                  docker-client-latest \
                  docker-common \
                  docker-latest \
                  docker-latest-logrotate \
                  docker-logrotate \
                  docker-engine
#2.需要的安装包
sudo yum install -y yum-utils

#3.设置镜像仓库
sudo yum-config-manager --add-repo 
https://download.docker.com/linux/centos/docker-ce.repo //默认国外

sudo yum-config-manager --add-repo 
http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo //建议阿里云

#更新软件包索引（建议）
yum makeecache fast

#4.安装docker相关的内容 docker-ce社区版
sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

#5.使用docker version 查看是否安装成功

#7.运行 hello-world
docker run hello-world

配置阿里云镜像加速器

#配置使用
#1.创建目录
sudo mkdir -p /etc/docker

#2.创建配置文件并写入
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://7us2yh4x.mirror.aliyuncs.com"]
}
EOF

#3.配置文件生效
sudo systemctl daemon-reload

#4.重启docker
sudo systemctl restart docker

如何卸载

#如何卸载

#1.卸载依赖
sudo yum remove docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin docker-ce-rootless-extras

#2.删除目录
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd

Docker命令

命令文档：https://docs.docker.com/reference/

镜像命令

docker images #查看所有本地镜像

docker search [name] #搜索镜像

docker pull [name] #下载镜像

docker rmi -f [ID] #根据ID删除镜像

容器命令

docker run [可选参数] image #新建容器并启动

docker ps #列出所有运行的容器

exit #容器退出并停止

Ctrl + P + Q #容器退出但不停止

docker rm [ID] #根据ID删除容器

docker start [ID] #启动容器

docker restart  [ID] #重启容器

docker stop [ID] #停止当前正在运行的容器

docker kill [ID] #强制停止当前容器

常用其它命令

docker run -d [name] #后台启动容器

docker logs -tf [-tail num] [ID] #查看日志

docker top [ID] #查看容器中进程信息

docker inspect [ID] #查看容器元数据

docker exec -it [ID] #进入当前正在运行的容器,开启一个新的终端

docker attach [ID] #进入容器正在执行的终端

docker cp 容器ID:容器内路径 目的主机路径 #从容器内拷贝文件到主机

练习

Docker运行nginx部署前端项目

思路

将nginx容器内的资源目录和配置目录映射到主机中的前端资源目录和配置目录，这样就可以实现nginx容器中的资源和配置与主机中的前端资源和配置一致，能够进行同步修改，即修改主机中的前端资源和nginx配置时，nginx容器中的资源和配置也能同步修改，避免每次修改资源和配置都需要进入容器内部，产生不必要的麻烦。

实现

1.拉取nginx镜像

docker pull nginx

2.复制nginx容器中的配置文件到主机的某个目录，例：/home/etc

#首先后台运行一个nginx容器
docker run --name nginx01 -d nginx
#进入该nginx容器
docker exec -it nginx01 /bin/bash
#查看资源目录和配置目录位置
whereis nginx

#nginx容器配置目录：/etc/nginx
#nginx容器资源目录：/usr/share/nginx

#将nginx容器中的配置文件复制到主机的/home/etc目录下
docker cp nginx01:/etc/nginx /home/etc
#停止并删除该镜像
docker stop nginx01
docker rm nginx01
#重新运行一个镜像并进行资源映射
docker run --name nginx02 -p 8084:80 -v /usr/frontProject:/usr/share/nginx/html -v /home/etc/nginx:/etc/nginx -d nginx
#命令解析
--name naginx02 #自定义容器名称为nginx02
-p 8084:80 #进行端口映射，将本机的8084端口映射到nginx容器的80端口
-v [主机资源目录:容器资源目录] #将主机的目录映射到容器的目录，两个目录双向绑定
-d nginx #设置nginx容器一直后台运行
#进入nginx配置文件目录
cd /home/etc/nginx/conf.d
#修改配置文件，按实际情况修改
vim default.conf 

#重启nginx镜像
docker restart nginx02

做完以上这些，即可通过访问 主机ip:主机端口即可访问到运行在nginx镜像上的前端项目。

容器数据卷

引入

个人理解

数据不应该放在容器中，如果容器被删除，数据也会丢失。比如nginx部署的前端资源和mysql数据库中的数据，所以需要将容器中产生的数据同步到本地，也就是目录的挂载，这就是卷技术。

使用方式

方式一：使用命令来挂载 -v

docker run -it -v 主机目录:容器目录

实战-安装mysql

#1.获取镜像
docker pull mysql:8.0 #指定版本为8.0，如不指定则为最新版
#2.启动镜像
docker run -d -p 3310:3306 -v /home/mysql/conf:/etc/mysql/conf.d -v /home/mysql/data:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=lihuicong_LHC_043930 --name mysql01 mysql:8.0
#命令解析
-d #后台启动
-p 3310:3306 #开放端口，主机3310端口映射到mysql容器3306端口
-v 主机目录:容器目录 #进行目录挂载，使主机目录中的资源与容器目录中的资源进行绑定，实现资源同步
-e #进行环境配置
MYSQL_ROOT_PASSWORD=**** #mysql第一次启动时设置密码
--name mysql01 #为容器命名
